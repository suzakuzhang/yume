# yume 调性宣言 · 阈 (The Threshold)

> 我作为 Claude 对这个产品的理解：yume 不是一个你"使用"的 app，而是一道你"跨过"的阈。
> 梦是脆弱的、半记得的、神圣的。界面要像**临睡的催眠态**（hypnagogic）——
> 在睡与醒之间，影像是软的，意义会增殖，几种声音可以同时低语而互不矛盾。

## 三脉交织

yume 的世界由三种血统编织而成，任何视觉/文案决策都应能落回其中之一：

- **墨** — 周公解梦 · 周易。东方的占梦传统：水墨、留白、笔触、阴阳。
- **星** — 塔罗 · 占星 · 西方神秘学。星座、金箔、轮、对应。
- **渊** — 精神分析。无意识、下降、阴影、未被照亮之物。

夜色是它们共同的底。墨在其上洇开，星在其上点亮，渊在其下沉默。

## 五条原则

1. **入而非用 (cross, don't operate)** — 进入 yume 像入睡：一次变暗、一次缓慢的绽开。不要仪表盘式的硬边信息密度。第一屏先让人静下来。
2. **软边 (soft edges)** — 模糊、辉光、淡入。没有东西"啪"地出现。慷慨的留白。衬线字、慢节奏、长行距。圆角与雾面。
3. **众声共在 (voices coexist)** — 四透镜不是标签页，而是**围拢在你的梦周围的在场者**，各有一种色与一枚印记（琥珀=弗洛伊德 / 靛=荣格 / 朱=周公 / 玉=占卜 / 银=科学哲学裁判）。它们一起低语，而非争夺。
4. **仪式弧 (the ritual arc)** — 每一次到访是一段小小的过关，给阈命名：
   **入夜** (录入意象与问题) → **凝象** (意象化作一幅画) → **众声** (四透镜立论与求同存异) → **问卜** (起卦 · 抽塔罗) → **拂晓** (综合 · 建议 · 自我探寻)。
5. **第二人称的低语 (the second-person whisper)** — 产品对"你"说话，像守在门口的引路者：通灵般，但温柔，绝不临床。不说"系统检测到"，要说"今夜，你带来了火"。

## 动效

- 全局：内容**淡入上浮**（fadeUp），缓 (0.9s, ease-out)。
- 关键元素：**呼吸**般的微辉光 (breathe)。
- 背景：星与雾极慢**漂移** (drift, 60s+)。
- 一律尊重 `prefers-reduced-motion`：关闭漂移与呼吸，仅保留淡入。

## 文案语域

慢、低、第二人称、意象优先。避免功能性词汇（"提交""记录成功"）。
"提交梦境" → "把今夜交付给阈"；"加载中" → "正在入梦…"；"暂无数据" → "阈是空的。从一个意象开始。"
